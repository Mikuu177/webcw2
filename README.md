# WebCW2 - Search Engine Tool

Coursework 2 for COMP/XJCO3011 Web Services and Web Data.

This project implements a command-line search engine for `https://quotes.toscrape.com/`.

## Architecture

```text
src/
  crawler.py  -> polite website crawler and HTML parsing
  indexer.py  -> tokenizer, inverted index, JSON persistence
  search.py   -> print/find logic, TF-IDF ranking, suggestions
  main.py     -> interactive command-line shell
tests/
  test_crawler.py
  test_indexer.py
  test_search.py
  test_cli.py
data/
  index.json  -> compiled index generated by build
```

## Features

- Polite crawler with at least 6 seconds between live HTTP requests.
- Inverted index with word frequency and positions per page.
- Persistent index saved to `data/index.json`.
- Interactive commands: `build`, `load`, `print`, `find`, `help`, `exit`.
- Ranked multi-word search and query suggestions.
- Automated tests for crawler, indexer, search, and CLI behavior.

## Usage

```bash
python -m src.main
```

Example commands inside the shell:

```text
build
load
print nonsense
find indifference
find good friends
find notarealword
exit
```

Command behavior:

- `build`: crawls the website, builds the inverted index, and saves `data/index.json`.
- `load`: loads the saved index file.
- `print <word>`: prints page-level frequency and positions for a word.
- `find <query>`: returns pages containing all query terms, ranked by TF-IDF score.
- `rank`: explains the ranking method.

## Data Structure and Algorithm Notes

The index stores a nested mapping:

```text
term -> url -> { frequency, positions }
```

This supports efficient lookup for a single term and set intersection for multi-word queries. Search ranking uses TF-IDF so pages with more occurrences score higher, while terms appearing on fewer pages become more distinctive.

For unknown words, the tool uses a small Levenshtein-distance suggestion helper. This improves the command-line user experience without adding unnecessary dependencies.

## Error Handling and Edge Cases

The tool handles:

- empty commands and empty queries,
- missing index files,
- unknown commands,
- missing search terms,
- non-existent words,
- network request failures during crawling.

## Submission Notes

The compiled index file is generated at `data/index.json` after running `build`.

GenAI was used as a planning and debugging aid. The video demonstration explains where AI suggestions helped, where they required correction, and how manual testing shaped the final implementation.

## GenAI Critical Evaluation Summary

GenAI was useful for comparing inverted-index data structures and identifying possible edge cases, but its first suggestions were too generic for the coursework requirements. I manually revised the design to store both frequency and positions, added mock-based tests to avoid real delays, and chose explainable TF-IDF ranking instead of opaque recommendations. This improved development speed while still requiring careful human validation.
